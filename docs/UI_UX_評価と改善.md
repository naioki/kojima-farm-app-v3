# UI/UX 評価と改善（仕事で使う際の手間削減）

## 評価の観点

- **きれい・分かりやすさ**: ラベル、タブ構成、メッセージの一貫性
- **手間が取られない**: デフォルト値の妥当性、余計な入力・クリックの有無、操作順の自然さ

---

## 良い点（現状）

| 項目 | 内容 |
|------|------|
| タブ構成 | 「画像解析」「メール」「未確定一覧」「台帳からPDF」「設定」で役割が分かれていて分かりやすい |
| サイドバー | APIキー・出荷日を常時表示し、メイン操作と分離されている |
| フィードバック | 成功時は「✅ ○件を…」、エラー時はメッセージ＋詳細 expander で原因が追いやすい |
| 解析結果 | 表で編集・店舗名は Selectbox で選択でき、納品データ・CSV・スプレッドシート追記まで一画面で完結 |
| 設定タブ | 店舗・品目・メール解析ルールを一箇所で管理できる |

---

## 実施した改善（手間削減）

1. **台帳シート名のデフォルト統一**  
   - 「台帳からPDF」「台帳に追記」のフォールバックを「シート1」→「台帳データ」に変更  
   - 多くの環境でシート名を変えずにそのまま使え、入力・失敗が減る  

2. **サイドバー「使い方」を折りたたみ**  
   - `st.expander("📋 使い方", expanded=False)` で初期は非表示  
   - APIキー・出荷日が目立ち、慣れた利用時は説明で画面を占有しない  

3. **トップの説明文を補足**  
   - 「画像アップロードだけでなく、メール・台帳連携・PDFまで一括」と分かるキャプションに変更  
   - 初回利用時に機能の全体像が伝わりやすい  

---

## 今後の検討（任意）

| 内容 | 効果 |
|------|------|
| スプレッドシートIDを Secrets で一元化 | タブごとにIDを入れ直す手間を削減 |
| 「解析結果あり」時に「ラベルを生成」へスクロール誘導 | 長い画面でボタンを見つけやすくする |
| メールタブで「設定を保存」をデフォルトオフのまま注意書き | パスワードを誤って保存しないよう案内 |
| 設定タブの「品目名管理」を「よく使う品目」と「その他」でグループ化 | 品目が多いときの探しやすさ向上 |

---

## まとめ

- **きれい・分かりやすさ**: タブとラベルで役割が明確で、メッセージも揃っている。  
- **手間**: 台帳まわりのデフォルト統一とサイドバー整理で、日常利用時の入力と確認が減るようにした。  
- 上記の「今後の検討」は、運用しながら必要に応じて追加するとよい。
